977A - Wrong Subtraction

Tutorial

977A - Wrong Subtraction

In this problem you just need to simulate the process described in the statement (i.e. kk times repeat the following operation: if n%10=0n%10=0 then n:=n/10n:=n/10 else n:=n1n:=n1) and print the result.

Solution (Vovuh)
#include <bits/stdc++.h>

using namespace std;

int main() {
#ifdef _DEBUG
	freopen("input.txt", "r", stdin);
//	freopen("output.txt", "w", stdout);
#endif

	int n, k;
	cin >> n >> k;
	
	for (int i = 0; i < k; ++i) {
		if (n % 10 == 0) n /= 10;
		else n--;
	}
	
	cout << n << endl;
	
	return 0;
}

977B - Two-gram

Tutorial

977B - Two-gram

There are at least two different approaches to this problem:

You can iterate over all substrings of ss of length 22 and calculate for each of them the number of its occurrences in ss (and try to update the result with the current substring).

Also you can iterate over all two-grams in the alphabet and do the same as in the aforementioned solution.

Solution (Vovuh)
#include <bits/stdc++.h>

using namespace std;

int main() {
#ifdef _DEBUG
	freopen("input.txt", "r", stdin);
//	freopen("output.txt", "w", stdout);
#endif

	int n;
	cin >> n;
	string s;
	cin >> s;
	
	int res = 0;
	string ans;
	for (int i = 0; i < n - 1; ++i) {
		int cur = 0;
		for (int j = 0; j < n - 1; ++j)
			if (s[j] == s[i] && s[j + 1] == s[i + 1])
				++cur;
		if (res < cur) {
			res = cur;
			ans = string(1, s[i]) + string(1, s[i + 1]);
		}
	}
	
	cout << ans << endl;
	
	return 0;
}

977C - Less or Equal

Tutorial

977C - Less or Equal

In this problem you can do the following thing: firstly, let's sort our array.

Let ansans will be the answer. Then you have two cases: if k=0k=0 then ans:=a01ans:=a01 otherwise ans:=ak1ans:=ak1 (for 0-indexed array).

Then you need to calculate the number of the elements of the array aa that are less than or equal to ansans. Let it be cntcnt. Then if ans<1ans<1 or cntkcntk then print "-1" otherwise print ansans.

Solution (Vovuh)
#include <bits/stdc++.h>

using namespace std;

int main() {
#ifdef _DEBUG
	freopen("input.txt", "r", stdin);
//	freopen("output.txt", "w", stdout);
#endif
	
	int n, k;
	scanf("%d %d", &n, &k);
	
	vector<int> a(n);
	for (int i = 0; i < n; ++i)
		scanf("%d", &a[i]);
	
	sort(a.begin(), a.end());
	
	int ans;
	
	if (k == 0) {
		ans = a[0] - 1;
	} else {
		ans = a[k - 1];
	}
	
	int cnt = 0;
	
	for (int i = 0; i < n; ++i)
		if (a[i] <= ans) ++cnt;
	
	if (cnt != k || !(1 <= ans && ans <= 1000 * 1000 * 1000)) {
		puts("-1");
		return 0;
	}
	
	printf("%d\n", ans);
	
	return 0;
}

977D - Divide by three, multiply by two

Tutorial

977D - Divide by three, multiply by two

Let deg3(x)deg3(x) be the maximum integer yy such that 3y|x3y|x (xx is divisible by 3y3y).

Our problem is to rearrange the given array in such a way that (easy to see it if we look at our operations) if it looks like ai1,ai2,,ainai1,ai2,,ain, then for each k[1..n1]k[1..n1] the next inequality will be satisfied: deg3(aik)deg3(aik+1)deg3(aik)deg3(aik+1). And if deg3(aik)=deg3(aik+1)deg3(aik)=deg3(aik+1) then numbers must be placed in increasing order (because of our operations). So we can store an array of pairs ansans, when ansi=(xi,yi)ansi=(xi,yi), xi=deg3(ai)xi=deg3(ai), yi=aiyi=ai. Then if we sort it in lexicographical order we can just print the second elements of the sorted array ansans.

Solution (eddy1021)
#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
int count3(LL x){
  int ret=0;
  while(x % 3 == 0){
    ret++;
    x /= 3;
  }
  return ret;
}
int n;
vector<pair<int,LL>> v;
int main(){
  cin>>n;
  v.resize(n);
  for(int i=0; i<n; i++){
    cin>>v[i].second;
    v[i].first=-count3(v[i].second);
  }
  sort(v.begin(), v.end());
  for(int i=0; i<n; i++)
    printf("%lld%c", v[i].second, " \n"[i + 1 == n]);
}

977E - Cyclic Components

Tutorial

977E - Cyclic Components

Let's solve this problem for each connected component of the given graph separately.

It is easy to see that the connected component is a cycle iff the degree of each its vertex equals to 22.

So the solution is to count the number of components such that every vertex in the component has degree 22.

The connected components of the graph can be easily found by simple dfs or bfs.

Solution (Vovuh)
#include <bits/stdc++.h>

using namespace std;

const int N = 200 * 1000 + 11;

int n, m;
int deg[N];
bool used[N];
vector<int> comp;
vector<int> g[N];

void dfs(int v) {
	used[v] = true;
	comp.push_back(v);
	
	for (auto to : g[v])
		if (!used[to])
			dfs(to);
}

int main() {
#ifdef _DEBUG
	freopen("input.txt", "r", stdin);
//	freopen("output.txt", "w", stdout);
#endif

	scanf("%d %d", &n, &m);
	for (int i = 0; i < m; ++i) {
		int x, y;
		scanf("%d %d", &x, &y);
		--x, --y;
		g[x].push_back(y);
		g[y].push_back(x);
		++deg[x];
		++deg[y];
	}
	
	int ans = 0;
	for (int i = 0; i < n; ++i) {
		if (!used[i]) {
			comp.clear();
			dfs(i);
			bool ok = true;
			for (auto v : comp) ok &= deg[v] == 2;
			if (ok) ++ans;
		}
	}
	
	printf("%d\n", ans);
	
	return 0;
}

977F - Consecutive Subsequence

Tutorial

977F - Consecutive Subsequence

Let dp[x]dp[x] be the answer for our problem if the last element of our subsequence equals to xx.

Then we have an easy solution: let's store dpdp as a "std::map" (C++) or "HashMap" (Java). Initially for each i[1..n]i[1..n] dp[ai]=0dp[ai]=0. Then let's iterate over all aiai in order of input and try to update dp[ai]dp[ai] with a dp[ai1]+1dp[ai1]+1 (dp[ai]=max(dp[ai],dp[ai1]+1)dp[ai]=max(dp[ai],dp[ai1]+1)).

Then the maximum element of dpdp will be our answer. Let it be ansans. Then let's find any aiai such that dp[ai]=ansdp[ai]=ans. Let it be lstlst. Then for restoring the answer we need to iterate over all elements of our array in reverse order and if the current element ak=lstak=lst then push kk to the array of positions of our subsequence and make lst:=lst1lst:=lst1.

Solution (Vovuh)
#include <bits/stdc++.h>

using namespace std;

int main() {
#ifdef _DEBUG
	freopen("input.txt", "r", stdin);
//	freopen("output.txt", "w", stdout);
#endif

	int n;
	scanf("%d", &n);
	vector<int> a(n);
	for (int i = 0; i < n; ++i)
		scanf("%d", &a[i]);
	
	map<int, int> dp;
	
	int ans = 0;
	int lst = 0;
	for (int i = 0; i < n; ++i) {
		int x = a[i];
		dp[x] = dp[x - 1] + 1;
		if (ans < dp[x]) {
			ans = dp[x];
			lst = x;
		}
	}
	
	vector<int> res;
	for (int i = n - 1; i >= 0; --i) {
		if (a[i] == lst) {
			res.push_back(i);
			--lst;
		}
	}
	reverse(res.begin(), res.end());
	
	printf("%d\n", ans);
	for (auto it : res)
		printf("%d ", it + 1);
	puts("");
	
	return 0;
}

For problem D you can view the problem as a directed graph where x maps to x*2 and x/3 if x|3.

Start point has indegree of 0.

Ignoring "extra" numbers in your dfs (i.e number not in your array) a dfs from the startpoint will always find a chain (guaranteed to exist) that is your answer.

void dfs(ll cur) { if (s[cur*2]) dfs(cur*2); if (cur%3==0 && s[cur/3]) dfs(cur/3); ans.push_back(cur); }

(you have to reverse ans)

    • If cnt < k then l = mid + 1. if cnt == k then this is our desire answer. If cnt > k r = mid-1; (l,r here is the left and right boundery, initially l = 1, r = 1e9)

And to be specific its the number in the solution above (between ans[k-1] and ans[k] 1 i think!). The fun part about binary search solution is that you dont have to worry about some corner case!
